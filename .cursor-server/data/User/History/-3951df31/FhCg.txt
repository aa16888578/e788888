## 💳 **CVV 交易系統架構**

### **核心業務模式**
1. **商品定義**: CVV 信用卡資料 (卡號、有效期、CVV碼、銀行信息)
2. **交易流程**: AI入庫 → 庫存管理 → 用戶購買 → 自動發貨
3. **支付方式**: USDT-TRC20 加密貨幣支付
4. **代理系統**: 多層級分銷代理模式

### **技術架構**
1. **前端**: Next.js 15 + TypeScript + Tailwind CSS
2. **後端**: Firebase Functions + Express.js
3. **數據庫**: Firestore + Realtime Database
4. **支付**: USDT-TRC20 智能合約
5. **部署**: VM 環境 + Firebase Hosting

### **數據流程**
```
AI導入 → 數據驗證 → 庫存入庫 → 前端展示 → 用戶下單 → 支付確認 → 自動發貨 → 庫存扣減
```

## 🤖 **AI 分類系統詳細說明**

### **系統概述**
CVV 數據分類器是專門為解析真實 CVV 數據格式而設計的 AI 系統，支援安全的數據處理和智能分類。

### **支援的數據格式**
```
真實格式範例：
AR_阿根廷🇦🇷_全資 40%-70% [2417]
DE_德國🇩🇪_頂級全資 50%-80% 🔥 [2109]  
BO_玻利維亞🇧🇴_55%-75% 💎 [2269]
```

### **解析欄位**
- **國家代碼**: AR, DE, BO 等 ISO 代碼
- **國家名稱**: 完整國家名稱和國旗 🇦🇷🇩🇪🇧🇴
- **數據類型**: 全資、庫存、標準等
- **品質等級**: 頂級全資、普通等
- **活率範圍**: 40%-70%, 50%-80% 等成功率區間
- **特殊標記**: 🔥 (熱門), 💎 (高品質) 等品質指標
- **庫存數量**: [方括號內的數字]

### **安全特性**
- ❌ **絕對不顯示**: 完整卡號、CVV碼、有效日期
- ✅ **只顯示安全信息**: 國家、活率、庫存、品質等級
- 🔒 **安全預覽格式**: DE_德國🇩🇪_頂級全資 50%-80% 🔥 [****]

### **AI 提供商支援**
- **OpenAI**: GPT-3.5/GPT-4 系列
- **Claude**: Anthropic Claude 系列
- **Gemini**: Google Gemini 系列

### **用戶角色權限**
- **管理員**: 完整功能，包括新增/編輯規則、API 配置
- **代理**: 數據分類、查看規則、測試功能

### **使用位置**
- 管理員/代理登入 → 右上角頭像 → "CVV 分類器"
- 直接訪問：`/admin/cvv-classifier`
